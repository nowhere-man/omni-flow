# OmniFlow Requirements

本文档只描述产品需求和用户可见行为。技术架构见 `docs/2.Architecture.md`，代码设计见 `docs/3.TechDesign.md`，验收标准见 `docs/4.Goal.md`。

---

## 1. 产品定位

- OmniFlow 是一款面向个人的多平台记账 App。
---

## 2. 平台范围

| 平台 | 支持 |
|------|------|
| iOS | 是 |
| macOS | 是 |
| Android | 是 |

- Android、iOS、macOS 均应提供符合各自平台习惯的界面体验。
- iOS 和 macOS 的导航、窗口布局、快捷键和信息密度应按平台分别设计。
- 目标体验是精美、成熟、符合各平台主流设计习惯。

### 2.1 主导航

- 手机端主导航包含：首页、统计、记账、搜索、更多。
- macOS 使用 Sidebar，主导航包含：首页、统计、记账、搜索、更多。
- 更多包含设置和模块化功能入口：导入、导出、规则、提醒、账本、账户、资产、分类管理、标签管理。

### 2.2 更多与设置

- 更多页第一版以 `examples/更多.PNG` 为视觉与交互参考进行复刻。
- 更多页顶部展示全局净资产概览和数据管理入口。
- 数据管理入口展示当前同步状态，并提供同步与备份恢复、导入、导出入口。
- 手机端更多页使用模块化图标入口组织设置、导入、导出、规则、提醒、账本、账户、资产、分类管理和标签管理。
- macOS 在更多节点下通过 Sidebar 子项或内容区模块入口进入相同功能。
- 设置页第一版以 `examples/设置.PNG` 为视觉与交互参考进行复刻。
- 设置首版仅支持应用锁、界面外观模式和数据管理入口；界面外观模式支持跟随系统、浅色和深色。

## 3. 首页

- 首页第一版以 `examples/首页1.PNG` 和 `examples/首页2.PNG` 为视觉与交互参考进行复刻。
- 首页支持账本范围选择，默认显示所有账本。
- 首页账本选择入口默认折叠；点击后展示“所有账本”和各个账本，用户选择后关闭列表。
- 首页日历、月度汇总和记账明细均按选中的账本范围展示；选择所有账本时汇总全部账本交易。
- 首页账本范围在重启 App 后保持；首次启动默认选择所有账本。
- 首页支持按月份切换日期范围。
- 首页每次启动自动显示当前月，不保留上次选择的月份。
- 首页使用上一个月、下一个月和月份选择控件切换月份。
- 首页展示所选月份的总支出、总收入、净收入。
- 首页提供日历控件，日历按天展示收支信息。
- 首页首版仅提供日历入口，不展示预算、趋势或建议入口。
- 首页日历支持筛选：全部、收入、支出。
- 日历筛选仅影响日历内每日收支信息的展示，不影响首页记账明细和日期明细面板。
- 月度汇总始终展示所选账本范围的总支出、总收入和净收入，不受日历筛选影响。
- 点击日历日期后，弹出日期明细面板，展示该日全部交易，不改变首页下方的记账明细。
- 日期明细面板初始继承首页账本范围；面板内切换账本范围仅影响该面板，不改变首页。
- 日期明细面板与首页记账明细复用同一明细模块，支持相同的账本选择、日合计、卡片或列表展示和交易编辑。
- 日期明细面板展示当日总支出、总收入及对应交易笔数；无交易日期显示空明细并可新增交易。
- 从日期明细面板新增交易时，交易日期预填为该日期；单账本范围预填该账本，所有账本范围不预填账本。
- 首页下方展示当前月份的全部记账明细，按日期倒序聚合。
- 每个日期分组展示当天合计支出或收入。
- 同一日期内的交易按交易时间倒序展示。
- 记账明细支持卡片形式和列表形式切换；首页和日期明细面板共用同一展示模式。

## 4. 基础记账

- 记账页第一版以 `examples/记账.PNG` 为视觉与交互参考进行复刻。
- 支持手动新增交易，字段包含：账本、账户、交易类型、日期、金额、一级分类、二级分类、标签、备注、来源。
- 交易类型枚举为收入、支出。
- 记账页支持支出、收入切换。
- 记账页以图标网格展示用户自定义的一级分类。
- 选中一级分类后，记账页展示该一级分类下的二级分类快捷选项。
- 记账页支持选择账户、标签、日期。
- 记账页支持备注输入。
- 记账页使用数字键盘录入金额，支持加、减、小数点、退格。
- 记账页支持「保存再记」和「完成」两个提交动作。
- 主导航、首页和日期明细面板中的新增入口均打开同一记账页；调用入口可预填交易日期和账本。
- 新建交易默认选择支出、当前时间和现金账户；仅在存在默认账本时自动选择默认账本。
- 一级分类必须选择，二级分类可不选择；选择二级分类时，交易关联该二级分类，否则关联一级分类。
- 保存再记后保留交易类型、账本、账户和日期，清空金额、分类、标签和备注；完成后关闭记账页。
- 每条交易必须归属于一个账本。
- 手动新增交易必须选择当前账本内的一级分类。
- 支持编辑已有交易的任意字段。
- 支持删除交易；删除后不应出现在默认列表、搜索和统计中。
- 支持将一笔交易标记为「不计入收支」，标记后该笔交易在统计图表中不参与计算。
- 不计入收支的交易仍按交易类型更新账户余额。
- 新增、编辑或删除交易后，关联账户余额按交易变更同步更新。
- 来源枚举为手动，支付宝导入、微信导入、美团导入、京东导入、抖音导入、建设银行导入。
- 青子记账导入不作为交易来源枚举值。
- 导入时解析到的商户名、交易对方、商品说明不作为独立交易字段；有值时全部合并保存到备注，没有值则不保存。

## 5. 多账本

- 支持创建、编辑、删除多个账本。
- 删除账本后，账本及其分类、标签、规则和交易不再出现在默认查询中，但保留在备份数据中。
- 首次启动时创建一个初始账本，但不自动设为默认账本。
- 默认账本是可选项。用户可指定或取消默认账本；存在默认账本时，未预填账本的记账页自动选择它。
- 不存在默认账本时，未预填账本的记账页不选择账本，用户必须在保存前选择账本。
- 删除默认账本后，默认账本自动取消。
- 没有可用账本时，首页显示创建账本引导；记账和导入必须先创建或选择账本。
- 支持账本设置名称和封面。
- 新建账本时自动复制系统预置的收入/支出一级分类模板。
- 支持对账本的收入/支出一级分类进行新增、编辑、删除。
---

## 6. 账户管理

- 账户是全局共享资源，不从属于任何账本，所有账本共用同一套账户池。
- 账户类型支持：现金、储蓄卡、信用卡、电子钱包、投资账户。
- 账户支持设置项目内置的统一 SVG 图标、名称、卡号、备注、余额、是否计入总资产。
- 支持创建、编辑、删除账户。
- 删除账户后，账户不再用于新增交易且不计入资产汇总；历史交易保留原账户并继续参与查询。
- 没有可用账户时，记账页显示创建账户引导，用户必须创建或选择账户后才能保存交易。
- 账户余额采用混合模式：记账时按交易自动更新余额，用户也可以手动校准当前余额；余额更新记录按天保留并记录增量。
- 手动校准只更新账户余额并保留校准记录，不新增交易。
- 初始账户默认创建现金、支付宝、微信、京东、美团、抖音、建设银行账户；默认账户类型为现金，使用项目内置的统一 SVG 图标，默认计入总资产。

### 6.1 资产

- 资产页展示所有计入总资产账户的余额、资产合计、负债合计和净资产。
- 计入总资产账户的余额大于零时计入资产，余额小于零时按绝对值计入负债；净资产等于资产合计减负债合计。账户类型不改变该计算规则。
- 资产页支持进入账户详情、创建账户和编辑账户。

### 6.2 货币

- 首版仅支持人民币。所有金额、余额、资产和统计结果均以人民币展示。
- 首版不支持多币种、汇率设置或金额换算。

## 7. 分类与标签

- 分类按账本独立，不全局共享。
- 分类为两级树结构：一级分类（父）和二级分类（子）。
- 一级分类必须设置图标，图标仅可从项目内置的统一 SVG 图标库中选择。
- 二级分类可不设置图标。
- 系统内置以下一级分类模板，新建账本时复制，用户可在账本内修改：
  - 支出类：餐饮、交通、日用、服务、娱乐、医疗、旅游、汽车、住房、数码、服饰。
  - 收入类：工资、理财、副业、奖金、红包。
- 二级分类默认不预置，用户首次创建后保存，后续可以直接使用，支持删除二级分类。
- 每个分类有固定的类型属性：支出或收入。
- 每条已入账交易必须关联一个当前账本内的分类。
- 删除分类后，该分类不再用于新增交易；历史交易保留原分类并继续参与搜索和统计。
- 一级分类存在未删除的二级分类时，必须先处理其二级分类后才能删除。
- 标签按账本独立，不全局共享。
- 标签为扁平多标签系统，一条交易可以打多个标签，也可以不打标签。
- 标签首次创建后保存，后续可以直接使用，支持删除标签。

## 8. 外部账单导入

### 8.1 支持的导入来源

| 来源 | 文件格式 | 编码/说明 |
|------|---------|-----------|
| 支付宝 | CSV | GBK |
| 微信 | XLSX | UTF-8 |
| 京东 | CSV | UTF-8 BOM |
| 美团 | CSV | UTF-8 BOM |
| 抖音 | CSV | UTF-8 BOM |
| 建设银行 | XLS | Excel 97-2003 |
| 青子记账 | JSON | UTF-8 |

### 8.2 通用导入规则
- 根据每个账单的内容解析，将账单的每条明细映射为本 App 记账明细的字段。
  - 用户从更多页进入导入，先选择目标账本，再通过系统文件选择器选择一个账单文件。
  - 系统根据文件格式和内容识别来源；无法唯一识别时由用户从支持的来源中选择。
  - 导入前必须指定账本。
  - 账户根据导入来源和明细内容自动解析并填写；青子记账导入默认使用现金账户。
  - 账本、账户、交易类型、日期、金额、一级分类必须填写；非青子记账导入还必须填写来源。
  - 二级分类、标签、备注根据内容解析，不存在则留空；商户名、交易对方、商品说明有值时合并保存到备注。
  - 如果日期没有时分秒，则统一按 `00:00:00`。
- 一级分类从选择的账本进行选择，先从已有的记录中根据关键字匹配，无法匹配则不填，让用户手动选择，选择后记住该分类，后续导入直接填入。
- 确认入账后，本次导入预览结束；再次导入同一文件时依赖去重能力避免重复入账。
- 最终交易必须保留来源平台和外部订单号，用于后续重复导入去重；青子记账导入不作为来源平台。

### 8.3 建设银行特殊规则
- 收支判断：金额正数 = 收入，金额负数 = 支出；摘要字段仅用于辅助验证。
- 时间精度仅到天，无时分秒，跨平台模糊去重时需特殊处理。

## 9. 规则引擎
- 规则按账本独立，不全局共享。
- 规则页显示当前账本的规则列表，支持切换账本、创建、编辑、删除和调整优先级。
- 一条规则包含名称、一个匹配条件、一个执行结果和优先级。
- 规则执行范围为导入预览项；已入账交易保持当前分类、账户、标签和排除状态。
- 规则执行动作限定在当前导入目标账本内。
- 匹配条件支持：备注包含关键字、交易类型等于收入或支出、来源平台等于指定来源。
- 执行结果支持：设置一级分类并可选设置二级分类、设为不计入收支、排除不入账。
- 规则按用户定义的优先级顺序执行，命中后短路。
- 支持规则的创建、读取、更新、删除。
- 用户可在导入预览页覆盖规则结果，最终以用户确认的预览结果入账。
---

## 10. 自动分类记忆

- 导入预览的初始分类填充顺序为：账本内规则 > 同账本历史分类记忆 > 空。
- 历史分类记忆的优先级低于用户自定义规则。
- 规则和历史分类记忆都未命中时，预览项分类为空，用户必须手动选择后才能入账。
- 用户在导入预览页手动选择或修改分类后，本次入账以用户选择为准。
- 用户手动选择的分类进入同账本历史分类记忆，后续相同或相似明细可自动填分类。
---

## 11. 去重引擎
- 绝对去重：相同来源平台 + 外部订单号的交易直接标记为重复，入账状态锁定为排除；青子记账导入不作为来源平台。
- 模糊去重：时间窗口、金额相等、备注文本相似度共同命中时标记为疑似重复。
- 普通来源的模糊去重时间窗口为交易时间前后 2 小时。
- 建设银行来源因时间仅精确到天，模糊去重窗口放宽为同日。
- 疑似重复项默认排除入账；用户手动确认保留后进入可入账状态。
---

## 12. 导入预览流程
- 用户上传文件后，系统自动完成解析、规则匹配和去重检测。
- 解析过程中显示进度和可读错误；解析失败不写入交易。
- 预览页展示完整的待入账明细列表。
- 重复项和疑似重复项有醒目标识。
- 用户可在预览页逐条编辑分类、账户、标签、备注、是否排除。
- 用户可在预览页批量操作：全选、反选、批量改分类、批量排除。
- 一键入账的前置条件：所有未排除预览项均已选择分类。
- 用户确认前展示本次入账的笔数、收入合计和支出合计；确认后一键入账，所有通过确认的明细写入账本。

## 13. 数据分析与图表

- 统计页第一版以 `examples/统计1.PNG`、`examples/统计2.PNG`、`examples/统计3.PNG` 和 `examples/统计4-点击查看账单表格.PNG` 为视觉与交互参考进行复刻。
- 统计页支持账本范围选择，默认显示所有账本；账本范围独立于首页并在重启 App 后保持。
- 统计页支持周、月、年和自定义范围；支持上一个、下一个时间范围和时间范围选择。
- 统计页每次启动默认显示当前月，不保留上次选择的时间范围。
- 统计页展示所选时间范围的收支趋势图、总支出、总收入和总结余，并展示与上一个等长时间范围的变化。
- 统计页展示支出或收入交易排行榜，按金额降序排列；用户可在支出和收入间切换。
- 统计页展示支出或收入分类占比，支持切换是否展开二级分类占比。
- 统计页提供标签分析入口，展示所选时间范围内按标签汇总的收支数据。
- 统计页提供账单表格入口；账单表格按年展示每月收入、支出和结余，以及全年合计。
- 统计结果自动排除不计入收支的交易。
- 各平台的数据口径必须一致，视觉和交互可符合各自平台习惯。
- 图表支持触点或鼠标选择、图例切换、时间范围切换、分类下钻和筛选联动。

## 14. 高级搜索与聚合

- 搜索页第一版以 `examples/搜索.PNG` 为视觉与交互参考进行复刻。
- 搜索默认范围为所有账本，支持筛选单个账本；搜索账本范围不与首页或统计页联动，也不在重启 App 后保持。
- 搜索支持关键词、交易类型、一级分类、二级分类、标签、账户、金额和日期任意组合筛选。
- 关键词匹配备注、一级分类名称、二级分类名称、账户名称和标签名称。
- 金额筛选支持不限、精确金额、最小金额和最大金额；日期筛选支持不限、某一天和时间范围。
- 手机端高级筛选默认展开，支持收起、展开和一键清除；macOS 使用常驻筛选栏。
- 输入关键词或调整任一筛选条件后更新搜索结果；全部筛选恢复默认时不展示结果。
- 搜索结果展示匹配交易列表，按交易时间倒序排列；点击交易可编辑。
- 搜索结果自动计算并展示筛选范围内的总收入和总支出。

## 15. 提醒与周期事项

- 提醒是独立功能，独立于账本、账户和交易。
- 提醒模块只产生提醒事件；交易、统计和账户余额由记账模块维护。
- 提醒参与多端同步。
- 提醒使用本地系统通知。
- 提醒分为两个子类型：
  - 还款提醒：金额可选，仅用于展示；支持固定还款日期或账单日后 N 天计算。
  - 订阅提醒：金额可选，用于展示和未来固定支出概览；支持每天、每周、每月、每年重复。
- 提醒支持简单重复规则。
- 提醒模块不生成待确认记账卡片，不与导入交易自动合并。

## 17. 青子记账互通

- 支持导入青子记账导出的 JSON 文件，导入后进入标准导入流水线。
- 支持将本项目数据导出为青子记账兼容的 JSON 格式。
- 导入导出页提供青子记账 JSON 导入与导出入口。
- 互通等级目标是长期双写：过渡期两个 App 同时使用，通过导入导出实现双向数据合并与增量同步。
- 字段映射规则：
  - `entry.value` → 金额
  - `entry.createDate` → 交易时间
  - `entry.content` → 备注
  - `category.type` → 交易类型
  - `entry.categoryID` → 关联分类名称
  - `entry.accountID` → 关联账户名称
  - `entry.markIDs` → 标签文本数组
  - `entry.excludeFromBudget = true` → 不计入收支
- 未映射字段（如位置信息、退款周期）在导入时记录警告并跳过，导入流程继续执行。
- 支持基于时间范围或重复检测的增量操作。

## 18. 数据同步

- 同步采用本地优先模型：无网络时所有功能正常工作，有网络时自动或手动同步。
- 同步目标限定为用户自有存储，仅用于保存与传输 App 数据。
- 支持 iCloud 和 WebDAV 作为同步目标。
- 首版同步数据不加密。
- 上传到同步目标的数据必须包含恢复 App 所需的全部业务数据和应用偏好。
- WebDAV 凭据、生物识别授权和通知权限不参与同步或恢复；恢复后由当前设备重新配置或授权。
- 用户删除 App 后重新安装，只要连接同一同步目标并恢复备份记录，恢复后的 App 数据必须与上传前一致。
- 用户可设置保留的最大备份记录数，超过上限后系统删除最旧备份记录。
- 最大备份记录数默认值为 10。
- 已配置同步目标且网络可用时，业务数据或应用偏好变更后延迟自动创建备份；连续变更合并为一次备份。用户可随时手动创建备份。
- 用户可查看备份记录，并选择一个备份记录恢复数据。
- 恢复为覆盖恢复：用户确认后，选定备份完整替换当前设备的全部可恢复状态，不与本地数据合并。
- 恢复确认界面提供创建当前备份的入口，用户可先备份当前状态再执行恢复。
